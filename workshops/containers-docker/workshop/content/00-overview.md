## What We're Doing

Containers are the fundamental packaging and runtime unit of cloud-native applications. Before you
can run anything on Kubernetes you need to understand how images are built, what makes a good
Dockerfile, and how to push images to a registry. This workshop covers all three.

## What You Will Learn

- What a container is and how it differs from a virtual machine
- How to read and write a Dockerfile
- How to build and run containers locally
- How multi-stage builds reduce image size and attack surface
- How to push an image to Harbor (the private registry on NKP)
